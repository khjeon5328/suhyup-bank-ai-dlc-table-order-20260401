# 테이블오더 서비스 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI + 관리자 UI + 백엔드 API + 데이터베이스)
- **Complexity**: Complex (멀티 매장, 역할 기반 권한, 실시간 통신, 대규모 동시 접속)
- **Depth**: Comprehensive

---

## 1. 기술 스택

| 구분 | 기술 |
|------|------|
| 백엔드 | Python FastAPI |
| 프론트엔드 | Vue.js |
| 데이터베이스 | MySQL |
| 실시간 통신 | Server-Sent Events (SSE) |
| 이미지 저장소 | 클라우드 스토리지 (S3 등) |
| 인증 | JWT 토큰 기반 |
| 배포 환경 | 클라우드 (AWS/Azure/GCP) |

---

## 2. 시스템 구성

### 2.1 주요 구성 요소
- **고객용 웹 UI**: 테이블 태블릿에서 동작하는 브라우저 기반 주문 화면 (Vue.js)
- **관리자용 웹 UI**: 매장 운영자가 사용하는 관리 대시보드 (Vue.js)
- **백엔드 API 서버**: 주문 처리 및 데이터 관리 (FastAPI)
- **데이터베이스**: 매장, 메뉴, 주문, 사용자 정보 저장 (MySQL)
- **클라우드 스토리지**: 메뉴 이미지 저장 (S3 등)

### 2.2 멀티 매장 구조
- 하나의 시스템에서 여러 매장을 관리
- 매장별 독립적인 메뉴, 테이블, 주문 데이터
- 매장별 역할 기반 관리자 계정 (점주, 매니저 등)

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- **세션 유효 기간: 무제한** (수동 변경 시까지 유지)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- **SSE를 통한 주문 상태 실시간 업데이트**
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 페이지네이션 또는 무한 스크롤

### 3.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한 (브루트포스 방지)

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **테이블 세션 처리**: 
  - 세션 시작 (첫 주문 시) 및 종료 (이용 완료)
  - 이용 완료 시: 주문 내역 → 과거 이력 이동, 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

#### FR-A05: 관리자 계정 및 역할 관리
- 매장당 여러 관리자 계정 지원
- 역할별 권한 구분 (점주, 매니저 등)
- 역할에 따른 기능 접근 제어

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 주문 표시: 2초 이내
- 동시 접속 500명 이상 지원
- 매장 20개 이상 운영 가능

### NFR-02: 보안
- JWT 토큰 기반 인증/인가
- 비밀번호 bcrypt 해싱
- HTTPS 강제 (TLS 1.2+)
- 입력값 검증 및 SQL injection 방지
- CORS 정책 적용
- HTTP 보안 헤더 (CSP, HSTS, X-Content-Type-Options 등)
- Rate limiting (브루트포스 방지)
- **보안 확장 규칙 (SECURITY-01 ~ SECURITY-15) 전체 적용**

### NFR-03: 확장성
- 멀티 매장 아키텍처
- 클라우드 배포 기반 수평 확장 가능

### NFR-04: 가용성
- 클라우드 인프라 기반 고가용성
- 에러 발생 시 graceful degradation

### NFR-05: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 기반 레이아웃
- 명확한 시각적 계층 구조

## 5. 제외 기능 (Out of Scope)

- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA/OTP)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 재고 관리, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도 API, 번역 API)
