# 테이블오더 서비스 - Application Design 통합 문서

---

## 1. 시스템 아키텍처 개요

멀티 매장 테이블오더 서비스로, 3-tier 아키텍처 기반의 모놀리식 구조입니다.

```
+------------------+     +-------------------+
| FE-CUSTOMER      |     | FE-ADMIN          |
| (Vue.js SPA)     |     | (Vue.js SPA)      |
+--------+---------+     +---------+---------+
         |      HTTPS/REST + SSE   |
         +------------+------------+
                      |
              +-------v--------+
              | FastAPI Server |
              +-------+--------+
                      |
         +------+-----+------+
         |      |             |
      +--v--+ +-v---+  +-----v-----+
      |MySQL| | S3  |  | SSE Mgr   |
      +-----+ +-----+  +-----------+
```

**기술 스택**: FastAPI (Python) + Vue.js + MySQL + S3 + SSE

## 2. 컴포넌트 요약

### 프론트엔드 (2개)
| ID | 컴포넌트 | 사용자 |
|----|---------|--------|
| FE-CUSTOMER | 고객용 웹 앱 | 매장 방문 고객 |
| FE-ADMIN | 관리자용 웹 앱 | 점주, 매니저 |

### 백엔드 모듈 (8개)
| ID | 모듈 | 핵심 책임 |
|----|------|----------|
| BE-AUTH | 인증/인가 | JWT 발급/검증, 역할 권한 검사 |
| BE-STORE | 매장 관리 | 매장 정보 조회 |
| BE-TABLE | 테이블 관리 | 테이블 설정, 세션 관리 |
| BE-MENU | 메뉴 관리 | 메뉴 CRUD, 카테고리 |
| BE-ORDER | 주문 관리 | 주문 CRUD, 상태 관리, 아카이브 |
| BE-SSE | 실시간 이벤트 | SSE 연결/브로드캐스트 |
| BE-USER | 사용자 관리 | 관리자 계정 CRUD |
| BE-IMAGE | 이미지 업로드 | S3 presigned URL |

### 인프라 (2개)
| ID | 컴포넌트 | 용도 |
|----|---------|------|
| DB-MYSQL | MySQL | 데이터 영속화 |
| STORAGE-S3 | 클라우드 스토리지 | 메뉴 이미지 저장 |

## 4. API 구조

모든 API는 `/api/v1/` 접두사 하에 매장 ID 기반으로 구성:

```
/api/v1/auth/          - 인증 (로그인)
/api/v1/stores/{id}/   - 매장 기반 리소스
  ├── tables/          - 테이블 관리
  ├── menus/           - 메뉴 관리
  ├── orders/          - 주문 관리
  ├── users/           - 사용자 관리
  ├── images/          - 이미지 업로드
  └── events/          - SSE 스트림
```

---

## 5. 핵심 데이터 흐름

1. **주문 생성**: 고객 → API → DB 저장 → SSE → 관리자 대시보드
2. **상태 변경**: 관리자 → API → DB 업데이트 → SSE → 고객 + 관리자
3. **이용 완료**: 관리자 → API → 주문 아카이브 → 테이블 리셋 → SSE → 고객
