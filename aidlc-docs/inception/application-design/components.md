# 테이블오더 서비스 - 컴포넌트 정의

## 1. 프론트엔드 컴포넌트

### FE-CUSTOMER: 고객용 웹 앱 (Vue.js)
**목적**: 테이블 태블릿에서 고객이 메뉴 조회, 장바구니 관리, 주문을 수행하는 SPA
**책임**:
- 태블릿 자동 로그인 및 세션 관리
- 카테고리별 메뉴 조회 및 상세 보기
- 장바구니 관리 (로컬 저장)
- 주문 생성 및 결과 표시
- 주문 내역 조회
- SSE를 통한 주문 상태 실시간 수신

### FE-ADMIN: 관리자용 웹 앱 (Vue.js)
**목적**: 매장 운영자(점주/매니저)가 주문 모니터링, 테이블/메뉴/계정을 관리하는 SPA
**책임**:
- 관리자 로그인 (JWT 기반)
- 실시간 주문 모니터링 대시보드 (SSE)
- 주문 상태 변경 및 주문 삭제 (점주만)
- 테이블 관리 (초기 설정, 이용 완료, 과거 내역)
- 메뉴 CRUD (점주만)
- 관리자 계정/역할 관리 (점주만)
- 역할 기반 UI 렌더링 (점주 vs 매니저)

## 2. 백엔드 컴포넌트 (FastAPI)

### BE-AUTH: 인증/인가 모듈
**목적**: 사용자 인증 및 역할 기반 접근 제어
**책임**:
- 관리자 로그인 (JWT 발급)
- 테이블 태블릿 로그인 (테이블 토큰 발급)
- JWT 토큰 검증 미들웨어
- 역할 기반 권한 검사 (점주/매니저/테이블)
- 브루트포스 방지 (로그인 시도 제한)
- 비밀번호 bcrypt 해싱

### BE-STORE: 매장 관리 모듈
**목적**: 매장 정보 관리
**책임**:
- 매장 정보 조회
- 시드 데이터 기반 매장 초기화

### BE-TABLE: 테이블 관리 모듈
**목적**: 테이블 설정 및 세션 라이프사이클 관리
**책임**:
- 테이블 초기 설정 (번호, 비밀번호)
- 테이블 세션 시작/종료
- 이용 완료 처리 (주문 → 과거 이력 이동, 리셋)
- 테이블 목록 조회

### BE-MENU: 메뉴 관리 모듈
**목적**: 메뉴 CRUD 및 카테고리 관리
**책임**:
- 메뉴 등록/조회/수정/삭제
- 카테고리별 메뉴 조회
- 메뉴 노출 순서 관리
- 이미지 URL 관리 (S3 업로드 연동)
- 입력값 검증 (필수 필드, 가격 범위)

### BE-ORDER: 주문 관리 모듈
**목적**: 주문 생성, 상태 관리, 이력 관리
**책임**:
- 주문 생성 (장바구니 → 주문)
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 주문 삭제 (점주 직권)
- 현재 세션 주문 조회
- 과거 주문 내역 조회 (날짜 필터)
- 주문 이벤트 발행 (SSE용)

### BE-SSE: 실시간 이벤트 모듈
**목적**: Server-Sent Events를 통한 실시간 데이터 전송
**책임**:
- SSE 연결 관리 (관리자/고객)
- 주문 생성/상태 변경/삭제 이벤트 브로드캐스트
- 매장별 이벤트 채널 관리
- 연결 해제 및 재연결 처리

### BE-USER: 사용자 관리 모듈
**목적**: 관리자 계정 및 역할 관리
**책임**:
- 관리자 계정 CRUD
- 역할 부여/변경 (점주/매니저)
- 계정 목록 조회

### BE-IMAGE: 이미지 업로드 모듈
**목적**: 메뉴 이미지 클라우드 스토리지 업로드
**책임**:
- S3 presigned URL 생성
- 이미지 업로드 처리
- 이미지 URL 반환

## 3. 데이터 컴포넌트

### DB-MYSQL: MySQL 데이터베이스
**목적**: 전체 서비스 데이터 영속화
**책임**:
- 매장, 테이블, 메뉴, 주문, 사용자, 세션 데이터 저장
- 트랜잭션 관리
- 인덱스 기반 쿼리 최적화

### STORAGE-S3: 클라우드 스토리지
**목적**: 메뉴 이미지 파일 저장
**책임**:
- 이미지 파일 저장 및 서빙
- presigned URL 기반 업로드/다운로드
