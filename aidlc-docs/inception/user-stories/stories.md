# 테이블오더 서비스 - User Stories

> 분류: Persona 기반 | 세분화: 중간 수준 | 수용 기준: 혼합 (GWT + 체크리스트) | 우선순위: MoSCoW

## 고객 (Customer) 스토리

### US-C01: 테이블 태블릿 자동 로그인
**As a** 고객,
**I want** 테이블 태블릿을 켜면 자동으로 로그인되어 바로 메뉴 화면이 표시되기를,
**So that** 별도의 로그인 절차 없이 즉시 주문을 시작할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 관리자가 태블릿에 매장 ID, 테이블 번호, 비밀번호를 1회 설정했을 때
- When: 태블릿 브라우저를 열면
- Then: 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다

- Given: 자동 로그인에 실패했을 때 (비밀번호 변경 등)
- When: 로그인 실패 응답을 받으면
- Then: 초기 설정 화면이 표시되어 재설정할 수 있다

**Persona**: 고객 (김민수)

---

### US-C02: 카테고리별 메뉴 조회
**As a** 고객,
**I want** 카테고리별로 분류된 메뉴를 카드 형태로 조회하기를,
**So that** 원하는 메뉴를 빠르게 찾아 선택할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 메뉴 화면이 기본(홈) 화면으로 표시된다
- ✅ 카테고리 탭으로 메뉴가 분류되어 표시된다
- ✅ 각 메뉴 카드에 메뉴명, 가격, 이미지가 표시된다
- ✅ 카테고리 간 빠른 이동이 가능하다
- ✅ 버튼/카드 터치 영역이 최소 44x44px이다

**Persona**: 고객 (김민수)

---

### US-C03: 메뉴 상세 정보 확인
**As a** 고객,
**I want** 메뉴 카드를 탭하면 상세 정보(설명, 큰 이미지)를 확인하기를,
**So that** 충분한 정보를 바탕으로 메뉴를 선택할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 메뉴 카드 탭 시 상세 정보 화면/모달이 표시된다
- ✅ 메뉴명, 가격, 설명, 이미지가 모두 표시된다
- ✅ 상세 화면에서 장바구니에 추가할 수 있다

**Persona**: 고객 (김민수)

---

### US-C04: 장바구니에 메뉴 추가
**As a** 고객,
**I want** 원하는 메뉴를 장바구니에 추가하기를,
**So that** 여러 메뉴를 모아서 한번에 주문할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 고객이 메뉴 화면에서 메뉴를 선택했을 때
- When: "추가" 버튼을 탭하면
- Then: 해당 메뉴가 장바구니에 추가되고 장바구니 아이콘에 수량이 표시된다

- Given: 이미 장바구니에 있는 메뉴를 다시 추가할 때
- When: 같은 메뉴의 "추가" 버튼을 탭하면
- Then: 해당 메뉴의 수량이 1 증가한다

**Persona**: 고객 (김민수)

---

### US-C05: 장바구니 수량 조절 및 삭제
**As a** 고객,
**I want** 장바구니에서 메뉴 수량을 조절하거나 삭제하기를,
**So that** 주문 전에 원하는 대로 수정할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 수량 증가/감소 버튼으로 수량 조절 가능
- ✅ 수량 1에서 감소 시 해당 메뉴 삭제 (또는 삭제 확인)
- ✅ 개별 메뉴 삭제 가능
- ✅ 장바구니 전체 비우기 가능
- ✅ 총 금액이 실시간으로 재계산된다
- ✅ 페이지 새로고침 후에도 장바구니가 유지된다 (로컬 저장)

**Persona**: 고객 (김민수)

---

### US-C06: 주문 생성
**As a** 고객,
**I want** 장바구니의 메뉴를 확인하고 주문을 확정하기를,
**So that** 실제 주문이 매장에 전달된다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 고객이 장바구니에 1개 이상의 메뉴가 있을 때
- When: 주문 내역을 확인하고 "주문하기" 버튼을 탭하면
- Then: 주문이 서버에 전송되고, 주문 번호가 표시된다

- Given: 주문이 성공했을 때
- When: 주문 완료 화면이 표시되면
- Then: 5초 후 메뉴 화면으로 자동 리다이렉트되고, 장바구니가 비워진다

- Given: 주문이 실패했을 때
- When: 서버 에러 응답을 받으면
- Then: 에러 메시지가 표시되고, 장바구니는 유지된다

**Persona**: 고객 (김민수)

---

### US-C07: 주문 내역 조회
**As a** 고객,
**I want** 현재 테이블 세션의 주문 내역을 조회하기를,
**So that** 이전에 주문한 내용과 상태를 확인할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- ✅ 주문 시간 순으로 정렬된다
- ✅ 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- ✅ 주문 상태(대기중/준비중/완료)가 표시된다
- ✅ 페이지네이션 또는 무한 스크롤 지원

**Persona**: 고객 (김민수)

---

### US-C08: 주문 상태 실시간 업데이트
**As a** 고객,
**I want** 주문 상태가 변경되면 실시간으로 화면에 반영되기를,
**So that** 페이지를 새로고침하지 않아도 현재 상태를 알 수 있다.

**Priority**: Should

**Acceptance Criteria**:
- Given: 고객이 주문 내역 화면을 보고 있을 때
- When: 관리자가 주문 상태를 변경하면 (대기중→준비중→완료)
- Then: SSE를 통해 해당 주문의 상태가 실시간으로 업데이트된다

**Persona**: 고객 (김민수)

## 점주 (Owner) 스토리

### US-O01: 관리자 로그인
**As a** 점주,
**I want** 매장 식별자와 계정 정보로 관리자 화면에 로그인하기를,
**So that** 매장 관리 기능에 접근할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 점주가 로그인 화면에서 매장 ID, 사용자명, 비밀번호를 입력했을 때
- When: "로그인" 버튼을 클릭하면
- Then: JWT 토큰이 발급되고 관리자 대시보드로 이동한다

- Given: 로그인에 성공한 후
- When: 16시간이 경과하면
- Then: 자동으로 로그아웃되고 로그인 화면으로 이동한다

- ✅ 브라우저 새로고침 시 세션이 유지된다
- ✅ 잘못된 자격 증명 시 에러 메시지가 표시된다
- ✅ 연속 로그인 실패 시 시도가 제한된다

**Persona**: 점주 (박영희)

---

### US-O02: 실시간 주문 모니터링
**As a** 점주,
**I want** 들어오는 주문을 실시간 대시보드에서 확인하기를,
**So that** 주문을 빠르게 파악하고 처리할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 테이블별 카드 형태의 그리드 레이아웃으로 표시된다
- ✅ 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- ✅ SSE를 통해 신규 주문이 2초 이내에 표시된다
- ✅ 신규 주문 시 시각적 강조(색상 변경, 애니메이션)가 적용된다
- ✅ 테이블별 필터링이 가능하다

**Persona**: 점주 (박영희)

---

### US-O03: 주문 상태 변경
**As a** 점주,
**I want** 주문의 상태를 변경하기를 (대기중→준비중→완료),
**So that** 주문 처리 진행 상황을 관리할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 점주가 주문 카드를 클릭했을 때
- When: 상태 변경 버튼을 클릭하면
- Then: 주문 상태가 다음 단계로 변경되고 화면에 반영된다

- ✅ 상태 전이: 대기중 → 준비중 → 완료 (순방향만)
- ✅ 상태 변경 시 고객 화면에도 실시간 반영 (SSE)

**Persona**: 점주 (박영희)

---

### US-O04: 주문 상세 보기
**As a** 점주,
**I want** 주문 카드를 클릭하면 전체 메뉴 목록을 상세히 보기를,
**So that** 주문 내용을 정확히 파악할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 주문 카드 클릭 시 상세 보기가 표시된다
- ✅ 테이블 번호, 주문 번호, 시각, 전체 메뉴 목록(메뉴명, 수량, 단가), 총 금액이 표시된다

**Persona**: 점주 (박영희)

---

### US-O05: 주문 삭제 (직권 수정)
**As a** 점주,
**I want** 특정 주문을 삭제하기를,
**So that** 잘못된 주문이나 취소 요청을 처리할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 점주가 주문 상세 보기에서 "삭제" 버튼을 클릭했을 때
- When: 확인 팝업에서 "확인"을 클릭하면
- Then: 주문이 즉시 삭제되고, 테이블 총 주문액이 재계산된다

- ✅ 삭제 전 확인 팝업이 표시된다
- ✅ 성공/실패 피드백이 표시된다
- ✅ 매니저 역할은 이 기능에 접근할 수 없다

**Persona**: 점주 (박영희)

**As a** 점주,
**I want** 테이블 번호와 비밀번호를 설정하기를,
**So that** 태블릿이 해당 테이블로 자동 로그인할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ 테이블 번호 및 비밀번호 입력 가능
- ✅ 설정 저장 시 세션이 생성된다
- ✅ 설정 완료 후 태블릿에서 자동 로그인이 활성화된다

**Persona**: 점주 (박영희)

---

### US-O07: 테이블 이용 완료 처리
**As a** 점주,
**I want** 고객이 떠난 테이블을 "이용 완료" 처리하기를,
**So that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- Given: 점주가 테이블 카드에서 "이용 완료" 버튼을 클릭했을 때
- When: 확인 팝업에서 "확인"을 클릭하면
- Then: 현재 세션의 주문 내역이 과거 이력으로 이동하고, 테이블 주문 목록과 총 주문액이 0으로 리셋된다

- ✅ 확인 팝업이 표시된다
- ✅ 성공/실패 피드백이 표시된다
- ✅ 새 고객의 첫 주문 시 새로운 세션이 시작된다

**Persona**: 점주 (박영희)

**As a** 점주,
**I want** 테이블별 과거 주문 내역을 조회하기를,

**Priority**: Must

**Acceptance Criteria**:
- ✅ "과거 내역" 버튼 클릭 시 과거 주문 목록이 표시된다
- ✅ 시간 역순으로 정렬된다
- ✅ 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- ✅ 날짜 필터링이 가능하다
- ✅ "닫기" 버튼으로 대시보드로 복귀한다

**Persona**: 점주 (박영희)

## 매니저 (Manager) 스토리

### US-M01: 매니저 로그인
**As a** 매니저,
**I want** 매장 식별자와 계정 정보로 관리자 화면에 로그인하기를,
**So that** 허용된 관리 기능에 접근할 수 있다.

**Priority**: Must

**Acceptance Criteria**:
- ✅ US-O01과 동일한 로그인 프로세스
- ✅ 로그인 후 매니저 권한에 맞는 메뉴만 표시된다
- ✅ 메뉴 관리, 주문 삭제, 계정 관리 메뉴는 표시되지 않는다

**Persona**: 매니저 (이준호)

---

### US-M02: 실시간 주문 모니터링 (매니저)
**As a** 매니저,
**I want** 들어오는 주문을 실시간으로 확인하고 상태를 변경하기를,
**So that** 주문을 빠르게 처리할 수 있다.
