# Unit 3: frontend-customer - 비즈니스 규칙

> 담당자: 국현

---

## BR-FC01: 자동 로그인
- localStorage에 토큰과 설정 정보가 있으면 자동 로그인 시도
- 토큰이 유효하면 메뉴 화면으로 이동
- 토큰이 무효하면 (401 응답) 설정 화면으로 이동
- 세션은 무제한 (서버에서 만료 관리)

## BR-FC02: 장바구니 로컬 저장
- 장바구니 데이터는 localStorage에 저장
- 페이지 새로고침 시에도 유지
- 서버에는 주문 확정 시에만 전송
- 세션 종료 이벤트(SSE) 수신 시 장바구니 자동 비우기

## BR-FC03: 장바구니 수량 규칙
- 최소 수량: 1
- 수량 1에서 감소 시 해당 항목 삭제
- 같은 메뉴 재추가 시 수량 +1
- 총 금액은 항목 변경 시 즉시 재계산

## BR-FC04: 주문 생성 조건
- 장바구니에 1개 이상의 항목이 있어야 주문 가능
- 빈 장바구니로 주문 시도 시 에러 메시지 표시
- 주문 중 중복 클릭 방지 (isLoading 상태)

## BR-FC05: 주문 성공 후 처리
- 주문 번호 표시
- 장바구니 자동 비우기 (localStorage 포함)
- 5초 카운트다운 후 메뉴 화면 자동 리다이렉트
- 카운트다운 중 사용자가 직접 이동 가능

## BR-FC06: 주문 실패 처리
- 에러 메시지 표시 (서버 응답 기반)
- 장바구니 유지 (비우지 않음)
- 재시도 가능

## BR-FC07: 주문 내역 필터링
- 현재 테이블 세션의 주문만 표시
- 이전 세션(아카이브된) 주문은 표시하지 않음
- 주문 시간 역순 정렬

## BR-FC08: SSE 연결 관리
- 인증 완료 후 자동 SSE 연결
- 연결 끊김 시 3초 간격으로 자동 재연결 (최대 10회)
- 재연결 실패 시 수동 새로고침 안내
- 페이지 이탈 시 연결 해제

## BR-FC09: 에러 표시 규칙
- 네트워크 에러: "서버에 연결할 수 없습니다. 잠시 후 다시 시도해 주세요."
- 인증 에러 (401): 설정 화면으로 자동 이동
- 서버 에러 (500): "일시적인 오류가 발생했습니다."
- 입력 검증 에러 (422): 서버 응답 메시지 표시

## BR-FC10: 터치 UI 규칙
- 모든 터치 가능 요소: 최소 44x44px
- 버튼 간 간격: 최소 8px
- 스크롤 영역: 터치 스와이프 지원
