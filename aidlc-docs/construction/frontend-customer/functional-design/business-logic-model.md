## 2. 상태 관리 흐름

### 인증 상태
```
초기 상태: token=null
  |
  +── loadFromStorage() → token 복원 또는 null 유지
  |
  +── login() 성공 → token 설정, storeId/tableId/sessionId 저장
  |
  +── 401 응답 수신 → token=null, /setup 이동
```

### 장바구니 상태
```
초기 상태: items=[] (localStorage에서 복원)
  |
  +── addItem(menu) → 기존 항목이면 quantity+1, 아니면 새 항목 추가
  |
  +── increaseQuantity(menuId) → quantity+1
  |
  +── decreaseQuantity(menuId) → quantity-1, 0이면 삭제
  |
  +── removeItem(menuId) → 항목 삭제
  |
  +── clearCart() → items=[], localStorage 클리어
  |
  모든 변경 시 → localStorage 자동 동기화, totalAmount 재계산
```

### 주문 상태
```
초기 상태: currentOrders=[]
  |
  +── fetchOrders(sessionId) → API 호출, 목록 갱신
  |
  +── createOrder(items) → API 호출, 성공 시 장바구니 비우기
  |
  +── SSE order_status_changed → 해당 주문 상태 업데이트
  |
  +── SSE session_ended → 주문 목록 클리어, 장바구니 비우기
```

## 3. 하단 네비게이션 구조

```
+--------------------------------------------------+
|                                                    |
|              [페이지 콘텐츠 영역]                   |
|                                                    |
+--------------------------------------------------+
| [🍽 메뉴]    [🛒 장바구니(N)]    [📋 주문내역]     |
+--------------------------------------------------+
```

- 메뉴: `/` (기본 화면)
- 장바구니: `/cart` (뱃지에 총 수량 표시)
- 주문내역: `/orders`

## 4. 폼 검증 규칙

### 초기 설정 폼
| 필드 | 검증 규칙 |
|------|----------|
| 매장 코드 | 필수, 1~50자 |
| 테이블 번호 | 필수, 양의 정수 |
| 비밀번호 | 필수, 1자 이상 |

### 주문 생성
| 항목 | 검증 규칙 |
|------|----------|
| 장바구니 | 1개 이상 항목 필수 |
| 수량 | 각 항목 1 이상 |
