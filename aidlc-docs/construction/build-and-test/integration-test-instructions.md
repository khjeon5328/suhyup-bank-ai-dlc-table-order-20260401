## 1. 사전 요구사항

- 백엔드 서버 실행 중 (`http://localhost:8000`)
- 데이터베이스 초기화 완료 (테스트 데이터 포함)
- frontend-admin 개발 서버 실행 중 (`http://localhost:3001`)

## 2. 테스트 시나리오 (5개)

### 시나리오 1: 인증 플로우 (Auth Flow)

**목적**: 로그인 → 토큰 발급 → API 호출 → 토큰 갱신 → 로그아웃 전체 플로우 검증

| 단계 | 동작 | 예상 결과 |
|------|------|----------|
| 1 | 로그인 페이지에서 매장 식별자, 사용자명, 비밀번호 입력 후 로그인 | 대시보드로 이동, Access Token 발급 |
| 2 | 대시보드에서 테이블 목록 API 호출 | Authorization 헤더에 토큰 포함, 200 응답 |
| 3 | Access Token 만료 후 API 호출 | 401 → 자동 토큰 갱신 → 원래 요청 재시도 성공 |
| 4 | 로그아웃 버튼 클릭 | 로그인 페이지로 이동, 토큰 초기화 |
| 5 | 브라우저 새로고침 | 로그인 페이지 표시 (Access Token 메모리 소멸) |

### 시나리오 2: SSE 실시간 모니터링 (SSE Monitoring)

**목적**: SSE 연결 → 실시간 이벤트 수신 → UI 반영 검증

| 단계 | 동작 | 예상 결과 |
|------|------|----------|
| 1 | 대시보드 진입 | SSE 연결 수립, SSEStatusIndicator "실시간 연결" 표시 |
| 2 | 고객이 주문 생성 (외부) | order_created 이벤트 수신, 해당 테이블 카드에 NEW 뱃지 표시 |
| 3 | 테이블 카드 클릭 | NEW 뱃지 제거, 주문 상세 모달에 신규 주문 표시 |
| 4 | 주문 상태 변경 (preparing) | order_status_changed 이벤트 발생, 상태 뱃지 업데이트 |
| 5 | SSE 연결 끊김 시뮬레이션 | 자동 재연결 시도, SSEStatusIndicator 상태 변경 |

### 시나리오 3: RBAC (역할 기반 접근 제어)

**목적**: owner/manager 역할별 기능 접근 제한 검증

| 단계 | 동작 | 예상 결과 |
|------|------|----------|
| 1 | owner 계정으로 로그인 | Sidebar에 대시보드, 메뉴 관리, 사용자 관리 모두 표시 |
| 2 | `/menus` 페이지 접근 | 메뉴 관리 페이지 정상 표시 |
| 3 | `/users` 페이지 접근 | 사용자 관리 페이지 정상 표시 |
| 4 | 주문 상세에서 삭제 버튼 | 삭제 버튼 표시, 삭제 기능 동작 |
| 5 | manager 계정으로 로그인 | Sidebar에 대시보드만 표시 |
| 6 | URL 직접 입력으로 `/menus` 접근 | 대시보드로 리다이렉트 |
| 7 | 주문 상세에서 삭제 버튼 | 삭제 버튼 미표시 |

### 시나리오 4: 메뉴 CRUD (Menu Management)

**목적**: 카테고리/메뉴 생성 → 수정 → 순서 변경 → 삭제 전체 플로우 검증

| 단계 | 동작 | 예상 결과 |
|------|------|----------|
| 1 | 카테고리 추가 ("음료") | 카테고리 목록에 "음료" 추가 |
| 2 | 메뉴 추가 (이름, 가격, 설명, 이미지) | 메뉴 목록에 신규 메뉴 표시, 이미지 S3 업로드 |
| 3 | 메뉴 수정 (가격 변경) | 수정된 가격 반영 |
| 4 | 카테고리 순서 drag-and-drop | 순서 변경 API 호출, 목록 재정렬 |
| 5 | 메뉴 삭제 | 확인 팝업 → 삭제 → 목록에서 제거 |

### 시나리오 5: 테이블 관리 (Table Management)

**목적**: 테이블 설정 → 주문 처리 → 이용 완료 → 과거 내역 조회 플로우 검증

| 단계 | 동작 | 예상 결과 |
|------|------|----------|
| 1 | 빈 테이블에서 "설정" 버튼 클릭 | TableSetupDialog 표시 |
| 2 | 테이블 번호, 비밀번호 입력 후 저장 | 테이블 세션 생성, 카드 상태 업데이트 |
| 3 | 주문 상태 변경 (pending → preparing → completed) | 상태 뱃지 및 버튼 업데이트 |
| 4 | "이용 완료" 버튼 클릭 | 확인 팝업 → 세션 종료 → 주문 목록 초기화 |
| 5 | "과거 내역" 버튼 클릭 | OrderHistoryPanel 표시, 이전 세션 주문 목록 |
| 6 | 날짜 필터 적용 | 필터된 과거 주문 목록 표시 |
