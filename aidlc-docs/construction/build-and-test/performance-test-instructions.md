# Performance Test Instructions - 테이블오더 서비스

## 성능 요구사항
- SSE 주문 표시: 2초 이내
- 동시 접속: 500명 이상
- 매장 20개 이상 운영

## 테스트 도구
- k6 (https://k6.io) 또는 Apache JMeter

## 테스트 시나리오

### 1. 메뉴 조회 부하 테스트
```
- 동시 사용자: 100 → 300 → 500
- 요청: GET /api/v1/stores/{id}/menus
- 목표: 평균 응답 시간 < 500ms, 에러율 < 1%
```

### 2. 주문 생성 부하 테스트
```
- 동시 사용자: 50 → 100 → 200
- 요청: POST /api/v1/stores/{id}/orders
- 목표: 평균 응답 시간 < 1000ms, 에러율 < 1%
```

### 3. SSE 동시 연결 테스트
```
- 동시 SSE 연결: 100 → 300 → 500
- 이벤트 발행 후 수신까지 시간 측정
- 목표: 이벤트 수신 < 2초
```

### 4. 혼합 시나리오
```
- 메뉴 조회 70% + 주문 생성 20% + 주문 내역 조회 10%
- 동시 사용자: 500
- 테스트 시간: 10분
- 목표: p95 응답 시간 < 2초
```
