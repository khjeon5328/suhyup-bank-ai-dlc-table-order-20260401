## 1. 역할 기반 접근 제어 (RBAC)

### BR-AUTH-01: 역할별 권한 매트릭스

| 기능 | 점주 (Owner) | 매니저 (Manager) |
|------|:-----------:|:---------------:|
| 대시보드 (주문 모니터링) | ✅ | ✅ |
| 주문 상태 변경 | ✅ | ✅ |
| 주문 상세 보기 | ✅ | ✅ |
| 주문 삭제 | ✅ | ❌ |
| 테이블 초기 설정 | ✅ | ❌ |
| 테이블 이용 완료 | ✅ | ✅ |
| 과거 주문 내역 조회 | ✅ | ✅ |
| 메뉴 관리 (CRUD) | ✅ | ❌ |
| 카테고리 관리 | ✅ | ❌ |
| 관리자 계정 관리 | ✅ | ❌ |

### BR-AUTH-02: 네비게이션 메뉴 렌더링
- 점주: 대시보드, 메뉴 관리, 사용자 관리 메뉴 모두 표시
- 매니저: 대시보드 메뉴만 표시
- 역할에 따라 Sidebar 메뉴 항목을 동적으로 렌더링

### BR-AUTH-03: 라우트 가드
- `/menus` 경로: owner 역할만 접근 가능
- `/users` 경로: owner 역할만 접근 가능
- 권한 없는 경로 접근 시 대시보드(`/`)로 리다이렉트
- 미인증 상태에서 모든 경로 접근 시 `/login`으로 리다이렉트

### BR-AUTH-04: UI 요소 가시성
- 주문 삭제 버튼: owner만 표시
- 테이블 설정 버튼: owner만 표시
- `v-if="isOwner"` 디렉티브로 조건부 렌더링

## 2. 인증 규칙

### BR-AUTH-05: 로그인 프로세스
- 매장 식별자 + 사용자명 + 비밀번호 입력
- 로그인 성공 시: Access Token 수신 → Pinia 메모리 저장, Refresh Token → httpOnly Cookie (서버 설정)
- 로그인 성공 후 대시보드(`/`)로 리다이렉트
- 로그인 폼 유효성 검사: 모든 필드 필수, 빈 값 제출 불가

### BR-AUTH-06: 세션 관리
- Access Token: Pinia store 메모리에 저장 (XSS 방지)
- Refresh Token: httpOnly Cookie로 관리 (클라이언트 접근 불가)
- Access Token 만료 시: Refresh Token으로 자동 갱신
- Refresh Token 만료 시: 로그인 화면으로 리다이렉트
- 브라우저 새로고침 시: Refresh Token Cookie로 Access Token 재발급
- 세션 유효 기간: 16시간 (서버 설정)

### BR-AUTH-07: 로그인 실패 처리
- 잘못된 자격 증명: "매장 정보 또는 로그인 정보가 올바르지 않습니다" 메시지
- 서버 에러: "서버에 연결할 수 없습니다" 메시지
- 연속 실패 시 서버 측 Rate Limiting 적용 (클라이언트는 에러 메시지만 표시)

## 3. 주문 상태 규칙

### BR-ORDER-01: 상태 전이 규칙
```
pending (대기중) → preparing (준비중) → completed (완료)
```
- 순방향 전이만 허용 (역방향 불가)
- 각 상태 변경은 서버 API 호출 후 반영
- 상태 변경 성공 시 SSE로 고객 화면에도 실시간 반영

### BR-ORDER-02: 상태별 UI 색상
| 상태 | 색상 | Element Plus 타입 |
|------|------|------------------|
| pending (대기중) | 주황색 | warning |
| preparing (준비중) | 파란색 | primary |
| completed (완료) | 초록색 | success |

### BR-ORDER-03: 주문 삭제 규칙
- 점주(owner)만 삭제 가능
- 삭제 전 확인 다이얼로그 표시 (ElMessageBox.confirm)
- 삭제 성공 시: 테이블 카드의 총 주문액 재계산, 주문 목록에서 제거
- 삭제 실패 시: 에러 메시지 표시

### BR-ORDER-04: 주문 모니터링 표시 규칙
- 대시보드에 모든 테이블을 카드 형태로 표시
- 각 테이블 카드: 테이블 번호, 총 주문액, 상태별 주문 수, 최신 주문 미리보기
- 신규 주문 수신 시: 해당 테이블 카드에 pulse 애니메이션 + "NEW" 뱃지
- 주문이 없는 테이블도 빈 카드로 표시

## 4. 테이블 세션 규칙

### BR-TABLE-01: 테이블 초기 설정
- 점주(owner)만 설정 가능
- 테이블 번호(숫자)와 비밀번호 입력
- 설정 완료 시 세션 생성 → 태블릿 자동 로그인 활성화

### BR-TABLE-02: 이용 완료 처리
- 점주, 매니저 모두 가능
- 확인 다이얼로그 표시 후 처리
- 처리 시: 현재 세션 주문 → 과거 이력 이동, 주문 목록/총액 리셋
- 새 고객의 첫 주문 시 새로운 세션 자동 시작

### BR-TABLE-03: 테이블 카드 표시
- 활성 세션이 있는 테이블: 주문 정보 표시
- 활성 세션이 없는 테이블: "주문 없음" 상태로 빈 카드 표시
- 카드 클릭 시 해당 테이블의 주문 상세 모달 열기

---

## 5. 과거 내역 규칙

### BR-HISTORY-01: 과거 주문 내역 조회
- 테이블 카드에서 "과거 내역" 버튼 클릭 시 사이드 패널 열기
- 시간 역순 정렬
- 날짜 필터(DateRange) 지원
- 각 내역: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
- "닫기" 버튼으로 패널 닫기

## 6. 대시보드 규칙

### BR-DASH-01: 테이블 그리드 표시
- 매장의 모든 테이블을 그리드로 표시 (주문 없는 테이블 포함)
- 테이블 번호 순으로 정렬

### BR-DASH-02: 반응형 그리드 레이아웃
| 화면 크기 | 열 수 | 비고 |
|-----------|-------|------|
| PC (1280px+) | 4~5열 | 기본 레이아웃 |
| 태블릿 (768~1279px) | 2~3열 | 축소 레이아웃 |

## 7. 다국어(i18n) 규칙

### BR-I18N-01: 지원 언어
- 한국어 (ko) - 기본 언어
- 영어 (en)

### BR-I18N-02: 구현 방식
- vue-i18n 라이브러리 사용
- locale 파일 분리: `src/locales/ko.json`, `src/locales/en.json`
- 모든 UI 텍스트는 i18n 키로 관리 (하드코딩 금지)

### BR-I18N-03: 언어 전환
- TopBar에 LanguageSwitcher 컴포넌트 배치
- 선택한 언어는 localStorage에 저장하여 새로고침 시 유지
- 언어 변경 시 전체 UI 즉시 반영 (페이지 리로드 불필요)

## 8. 카테고리 관리 규칙

### BR-CATEGORY-01: 카테고리 CRUD
- 점주(owner)만 카테고리 생성/수정/삭제 가능
- 카테고리명: 필수, 1~50자
- 카테고리 삭제: 해당 카테고리에 메뉴가 없을 때만 가능
- 메뉴가 있는 카테고리 삭제 시도 시 에러 메시지 표시
- 카테고리 순서 변경: drag-and-drop 지원

---

## 9. 메뉴 관리 규칙

### BR-MENU-01: 메뉴 입력 유효성 검사
| 필드 | 규칙 |
|------|------|
| 메뉴명 | 필수, 1~100자 |
| 가격 | 필수, 0 이상 정수 |
| 설명 | 선택, 최대 500자 |
| 카테고리 | 필수, 기존 카테고리 선택 |
| 이미지 | 선택, JPG/PNG, 최대 5MB |

### BR-MENU-02: 이미지 업로드
- S3 클라우드 스토리지에 업로드
- 업로드 전 파일 형식(JPG, PNG) 및 크기(5MB) 검증
- 업로드 성공 시 반환된 URL을 메뉴 데이터에 저장
- 업로드 실패 시 에러 메시지 표시, 메뉴 저장은 이미지 없이 가능

### BR-MENU-03: 메뉴 순서 변경
- drag-and-drop으로 메뉴 노출 순서 변경 (vuedraggable)
- 순서 변경 시 서버에 일괄 업데이트 요청
- 같은 카테고리 내에서만 순서 변경 가능

### BR-MENU-04: 메뉴 삭제
- 삭제 전 확인 다이얼로그 표시
- 삭제 성공/실패 피드백 표시

## 10. 사용자 관리 규칙

### BR-USER-01: 사용자 입력 유효성 검사
| 필드 | 규칙 |
|------|------|

- 자기 자신의 계정은 삭제 불가
- 삭제 전 확인 다이얼로그 표시

## 11. SSE 규칙

### BR-SSE-01: 연결 및 재연결
- 로그인 성공 후 자동 SSE 연결
- 연결 끊김 시 자동 재연결 (최대 3회, 3초 간격)
- 3회 재연결 실패 시 SSE 상태 표시기에 "연결 끊김" 표시
- 로그아웃 시 SSE 연결 해제

### BR-SSE-02: 이벤트별 UI 반응

| 이벤트 | UI 반응 |
|--------|---------|
| order_created | 해당 테이블 카드에 주문 추가, NEW 뱃지, pulse 애니메이션 |
| order_status_changed | 해당 주문의 상태 뱃지 업데이트 |
| order_deleted | 해당 주문 목록에서 제거, 총액 재계산 |

### BR-SSE-03: 시각적 알림
- 신규 주문: 테이블 카드 테두리 pulse 애니메이션 (3초간)
- NEW 뱃지: 해당 테이블 카드에 표시, 카드 클릭 시 제거
- SSE 연결 상태: TopBar에 SSEStatusIndicator로 표시 (연결됨/연결 끊김)

## 12. 에러 처리 규칙

### BR-ERROR-01: HTTP 상태별 처리

| HTTP 상태 | 처리 |
|-----------|------|
| 400 | 입력값 오류 메시지 표시 |
| 401 | Access Token 갱신 시도 → 실패 시 로그인 화면 이동 |
| 403 | "권한이 없습니다" 메시지 표시 |
| 404 | "요청한 리소스를 찾을 수 없습니다" 메시지 |
| 422 | 서버 유효성 검사 에러 메시지 표시 |
| 500 | "서버 오류가 발생했습니다" 메시지 |
| 네트워크 에러 | "서버에 연결할 수 없습니다" 메시지 |

### BR-ERROR-02: 사용자 피드백
- 성공 메시지: ElMessage.success (3초 자동 닫힘)
- 에러 메시지: ElMessage.error (5초 자동 닫힘)
- 경고 메시지: ElMessage.warning (4초 자동 닫힘)
- 확인 필요 작업: ElMessageBox.confirm 사용
