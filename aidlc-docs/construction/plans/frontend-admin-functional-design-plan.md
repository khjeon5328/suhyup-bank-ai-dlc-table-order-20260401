# Functional Design Plan - frontend-admin (관리자용 프론트엔드)

## 유닛 개요
- **유닛명**: frontend-admin
- **기술 스택**: Vue.js 3 + Pinia + Vue Router
- **할당 스토리**: 15개 (US-O01~O11, US-M01~M04)
- **역할**: 점주/매니저가 사용하는 관리자 SPA

---

## 실행 계획

### Phase A: 도메인 모델 및 상태 관리 설계
- [x] A1. 프론트엔드 도메인 엔티티 정의 (TypeScript 인터페이스/타입)
- [x] A2. Pinia 스토어 구조 설계 (auth, order, table, menu, user 스토어)
- [x] A3. API 응답/요청 DTO 타입 정의

### Phase B: 라우팅 및 페이지 구조 설계
- [x] B1. Vue Router 라우트 구조 정의 (역할 기반 가드 포함)
- [x] B2. 페이지(View) 컴포넌트 목록 및 계층 구조 정의
- [x] B3. 역할 기반 네비게이션 메뉴 구조 설계

### Phase C: 컴포넌트 설계
- [x] C1. 공통 UI 컴포넌트 정의 (Button, Modal, Card, Form 등)
- [x] C2. 기능별 컴포넌트 계층 구조 설계
- [x] C3. 컴포넌트별 Props/Emits/Slots 인터페이스 정의
- [x] C4. 폼 검증 규칙 정의

### Phase D: API 연동 및 실시간 통신 설계
- [x] D1. API 서비스 레이어 설계 (axios 인스턴스, 인터셉터)
- [x] D2. SSE 연결 관리 설계 (연결/재연결/이벤트 핸들링)
- [x] D3. 에러 핸들링 전략 정의

### Phase E: 비즈니스 규칙 정의
- [x] E1. 역할 기반 접근 제어 규칙 (점주 vs 매니저 UI 분기)
- [x] E2. 주문 상태 전이 규칙 (대기중→준비중→완료)
- [x] E3. 테이블 세션 라이프사이클 규칙
- [x] E4. 폼 입력 검증 비즈니스 규칙 (메뉴, 계정 등)

### Phase F: 사용자 인터랙션 플로우 정의
- [x] F1. 로그인/로그아웃 플로우
- [x] F2. 주문 모니터링 및 상태 변경 플로우
- [x] F3. 테이블 관리 플로우 (설정, 이용 완료, 과거 내역)
- [x] F4. 메뉴 관리 플로우 (CRUD + 이미지 업로드)
- [x] F5. 계정 관리 플로우
